// 02 — Machine Learning Models

Three models.
Two are blind to attacks.

The unsupervised models only learn normal behaviour. They have never seen a single attack packet — making them capable of detecting threats that have never existed before.

[RF]
Random Forest
Supervised — Baseline

Trained with labelled attack data. Acts as the gold standard comparison. Near-perfect performance across all modules demonstrates the problem is solvable — the challenge is doing it without labels.

Avg AUC: 0.9999
[IF]
Isolation Forest
Unsupervised — Zero-Day

Isolates anomalies by randomly partitioning the feature space. Trained only on benign traffic. Anomalies require fewer splits to isolate — detectable without any prior attack knowledge.

Avg AUC: 0.7079
[AE]
Autoencoder
Unsupervised — Zero-Day

A deep neural network that learns to reconstruct normal traffic. When attack traffic passes through, reconstruction error spikes — flagging the anomaly. Outperforms Isolation Forest on 4 of 5 modules.

Avg AUC: 0.7342

// 04 — Explainability

Why did it flag that?

SHAP (SHapley Additive exPlanations) reveals which network features drove each detection decision — making the system transparent for SOC analysts.

TOP 10 FEATURES — DDoS MODULE
Bwd Packet Length Mean 0.2407
Bwd Packet Length Max 0.2405
Bwd Packet Length Std 0.2097
Avg Bwd Segment Size 0.1967
Packet Length Std 0.1612
Packet Length Variance 0.1437
Packet Length Mean 0.1380
Flow IAT Max 0.1131
Max Packet Length 0.1041
Average Packet Size 0.0969
KEY INSIGHT

The top features are all related to backward packet length — the size of packets flowing back from the server to the client.

During a DDoS attack, the server sends near-identical small error responses to thousands of spoofed requests — making backward packet sizes unnaturally uniform and small.

The Isolation Forest learned this pattern purely from normal traffic deviation — without ever seeing a single DDoS packet during training.
